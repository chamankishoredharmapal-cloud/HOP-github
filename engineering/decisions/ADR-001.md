# ADR-001: Engineering Documentation Structure

## Status

Accepted

## Context

The HOP codebase lacked centralized engineering documentation, making it difficult to track audit progress, architectural decisions, and verification results. A structured documentation system was needed to govern Phase 1 of the Production Readiness Audit and serve as the foundation for future phases.

## Decision

Create an `engineering/` directory at the repository root with the following structure:
- `README.md` — entry point and navigation
- `roadmap.md` — phase planning
- `audit/` — per-phase audit artifacts
- `decisions/` — Architecture Decision Records
- `reports/` — supplementary reports
- `verification/` — verification protocols and results
- `templates/` — reusable templates for tickets, audits, verification

## Consequences

- All engineering documentation has a predictable location.
- New team members can navigate the engineering system via a single entry point.
- ADRs provide a permanent record of why decisions were made.
- Templates enforce consistency across tickets and audits.
